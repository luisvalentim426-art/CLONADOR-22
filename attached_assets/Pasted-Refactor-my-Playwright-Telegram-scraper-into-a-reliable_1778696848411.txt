Refactor my Playwright + Telegram scraper into a reliable Facebook Ads funnel crawler.

- It is not decoding Facebook redirect links (l.facebook.com/l.php?u=...)

GOAL:
Build a stable funnel crawler that always follows the REAL ad destination and ignores all irrelevant Facebook UI links.

REQUIREMENTS:

1. LINK EXTRACTION FIX
- Extract all <a> links from Ads Library pages
- Decode Facebook redirect links:
  - l.facebook.com/l.php?u= should be decoded using URLSearchParams "u"
- Remove ALL links that are:
  - facebook.com
  - l.facebook.com

- Only ONE funnel entry link must be selected per ad:
  - the first valid external link

  - ALWAYS navigate to it using Playwright page.goto()

4. SCREENSHOT RULE
- Take screenshot ONLY after navigating to the real landing page

- detect page type:
  - quiz (radio buttons, multi-step forms, "next/continue")
  - checkout (payment keywords)
  - VSL (video/iframe presence)
  - upsell/downsell (keywords)
- extract all internal funnel steps (same or related domain)
- follow up to max depth 5

6. OUTPUT
For each ad, send Telegram message containing:
- advertiser
- final landing URL
- funnel type
- number of steps detected
- screenshots per step (not only first page)

7. CRITICAL RULE
Do NOT use raw DOM links from Ads Library as final destinations.

Write production-ready TypeScript using Playwright and keep current Telegram integration.
